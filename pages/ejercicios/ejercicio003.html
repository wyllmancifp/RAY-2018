Evaluación Web personal

Ejercicio 3 (Individual)

Práctica de evaluación


Jornada de evaluación de la web personal de cada alumno, creada en los ejercicios 1 y 2. Se pedirá la entrega de los ejercicios completos y la realización de una modificación a realizar en clase. Todo ello subido en github, haciendo uso del servicio gratuito "GitHub Pages"


- Ejercicios 1 y 2 (5 puntos)
- Modificación (3 puntos)
- Claridad y orden de las estructuras y el código (2 puntos)

Requisitos de la entrega:

- Ejercicios 1 y 2
Se ha de entregar una página web personal usando la plantilla html-css proporcionada por el profesor (con los estilos css propios de cada alumno). Cada una de las subpáginas ha de tener su sección propia para cabecera, cuerpo y pié. La cabecera y el pie ha de ser común en todas las subpáginas.
Las cabeceras dispondrán de un texto o logo identificativo y un mapa de navegación [Inicio, Prácticas, Recursos]
La sección de inicio se enlazará con index.html siendo la página portada de la web (Ficha personal del alumno)
La sección de ejercios ha de enlazar con las subpáginas de cada uno de los enunciados y entregas de cada uno de ellos.
La sección de recursos ha de enlazar a las subpáginas de recursos html y recursos css creados en el ejercicio 2.

- Modificación
Se ha de modificar el mapa de navegación, añadiendo una sección principal "Videojuegos" con una subsección "Bricks", quedando con las siguientes secciones como mínimo:
Inicio,
Prácticas,
- Ejercicio 1
- Ejercicio 2
- Ejercicio 3
Recursos
- HTML5
- CSS3
Videojuegos
- Bricks

Se modificarán las cabeceras de todas las páginas y subpáginas para reflejar el nuevo mapa de navegación. La sección Videojuegos enlazará con una página índice "videojuegos.html" y la sección Bricks enlazará con la subpágina "videojuegos/bricks.html" a crear por el alumno.
La página videojuegos.html contendrá un índice de las subpáginas de juegos(bricks.html), imitando el estilo de la página "pracitcas.html".

La página bricks.html ha de enlazar el archivo de estilos "bricks.css" y el archivo de funcionalidades js "bricks.js". Su estructura básica seguirá los estilos del resto de la web (cabecera y pie común), el cuerpo de la web contendrá las estructuras necesarias para la ejecución del juego Bricks(Copiar el código del archivo "brickSection.html") definido en los archivos enlazados.

Se han de adaptar los estilos del archivo bricks.css para solucionar las posibles incompatibilidades con los estilos del resto de la web y, además:
- Modificar el div con identificador "gameBackground" para usar una imagen de fondo propia, en vez de un color sólido.
- Modificar el div con identificador "gameButtonsSec": [cambiar color de fondo y añadir un botón("Iniciar") más que lance la funcion "init()" en el evento de click]
- Adaptar los valores de profundidad en el eje z (z-index) para que la cabecera quede accesible
